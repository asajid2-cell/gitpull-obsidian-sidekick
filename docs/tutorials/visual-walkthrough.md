# Visual Walkthrough

These screenshots were retaken on a Pixel 9 Pro XL Android emulator at 1344x2992 after verifying the app respects Android safe areas.

## Setup

Open `gitpull`, sign in with GitHub or use the fallback token field, choose a vault folder, then save the configuration.

<img src="../assets/screenshots/01-setup-start.png" alt="Setup screen" width="360">

After sign-in, the repository browser can load repos available to your GitHub account.

<img src="../assets/screenshots/02-setup-repo-browser.png" alt="Repository browser" width="360">

## Pull

The Pull tab shows the selected repository, branch, destination folder, last pull time, and the main Pull action.

<img src="../assets/screenshots/03-pull-ready.png" alt="Pull ready" width="360">

During a refresh, the app disables the button and shows pull progress.

<img src="../assets/screenshots/04-pull-progress.png" alt="Pull in progress" width="360">

When the snapshot is ready, Obsidian Mobile can open the selected folder as the vault.

<img src="../assets/screenshots/05-pull-success.png" alt="Pull success" width="360">

## PDFs

The PDFs tab indexes pulled PDF files so they can be opened through Android's standard open-with flow.

<img src="../assets/screenshots/06-pdfs-list.png" alt="PDF list" width="360">

## Settings

Settings let you update the repository, reload repository choices, change the destination folder, manage GitHub sign-in or token fallback, and clear the local snapshot.

<img src="../assets/screenshots/07-settings-repository.png" alt="Settings repository" width="360">

<img src="../assets/screenshots/08-settings-safety.png" alt="Settings safety actions" width="360">
